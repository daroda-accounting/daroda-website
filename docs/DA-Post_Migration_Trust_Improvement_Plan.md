# Daroda Accounting — Post‑Migration Trust Improvement Plan

## Purpose

This document outlines practical steps to improve website trust, security posture, search visibility, and technical reputation **after migration away from Wix to the new custom platform.**

The goal is not only to maintain current trust signals, but to measurably strengthen them over time.

---

## Success Definition

Post‑migration success will mean:

- No browser security warnings
- Clean reputation scans across major security vendors
- Improved technical security header score
- Stable uptime and fast page performance
- Gradual increase in search visibility and discovery
- Strong professional credibility signals for prospective clients

---

## Phase 1 — Immediate Post‑Launch Trust Validation (Week 1)

### 1. Verify HTTPS and Certificate Health

- Confirm SSL certificate is valid and auto‑renewing
- Confirm HTTP → HTTPS redirect enforced
- Confirm no mixed‑content warnings

Tools:

- SSL Labs
- Browser DevTools Security Panel

### 2. Run Reputation Scans

Confirm clean status across:

- Google Safe Browsing
- VirusTotal
- ScamAdviser
- DNSFilter

Document baseline screenshots for future comparison.

### 3. Verify Search Indexing

- Submit sitemap in Google Search Console
- Request indexing for homepage and service pages
- Confirm no manual penalties or crawl errors

---

## Phase 2 — Security Hardening Improvements (Weeks 1–3)

### 4. Implement Modern Security Headers

Improve score from previous "D" baseline by adding:

- Content‑Security‑Policy
- X‑Frame‑Options
- Referrer‑Policy
- Permissions‑Policy

Goal:

Achieve **B or higher** on securityheaders.com.

### 5. Improve Cookie Security

Ensure:

- HttpOnly where applicable
- Secure flag enabled
- SameSite strategy appropriate

### 6. Reduce Third‑Party Surface Area

- Limit unnecessary external scripts
- Prefer self‑hosted assets where practical

---

## Phase 3 — Performance & Reliability Trust Signals (Weeks 2–4)

### 7. Establish Monitoring

Implement uptime monitoring:

- UptimeRobot / BetterStack / similar

Goal:

- Receive alerts for outages
- Maintain >99.9% uptime

### 8. Performance Benchmarking

Measure via:

- Google PageSpeed Insights
- WebPageTest

Track:

- Core Web Vitals
- Load time improvements vs Wix baseline

---

## Phase 4 — Reputation & Authority Growth (Months 1–6)

### 9. Structured SEO Improvements

- Add structured metadata
- Improve service page content depth
- Optimize headings and internal linking

### 10. Professional Trust Signals

Enhance perceived credibility through:

- Testimonials / reviews
- Clear contact information
- Professional imagery and branding consistency

### 11. Backlink & Discovery Strategy

Encourage:

- Local directory listings
- Accounting association references
- Speaking / networking mentions that include website link

---

## Phase 5 — Ongoing Trust Maintenance

### 12. Quarterly Trust Audit

Re‑run:

- Security header scan
- Reputation checks
- Indexing validation

Document trends over time.

### 13. Dependency Updates

- Keep hosting platform updated
- Maintain build tooling and libraries

### 14. Incident Response Preparedness

Maintain ability to:

- Roll back deployment
- Restore backups
- Temporarily disable affected features

---

## Conclusion

Website trust is not a one‑time milestone but an ongoing practice.

This plan ensures the new Daroda Accounting website:

- launches safely
- strengthens technical credibility
- supports long‑term business growth
- remains maintainable for future developers

By tracking these improvements deliberately, the migration becomes not just a redesign, but a measurable upgrade in professionalism, reliability, and client confidence.
