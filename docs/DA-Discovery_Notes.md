# Daroda Accounting — Discovery & Objectives Notes

---

## SECTION 1 — Business Objectives

**A. Main Business Goal**

- Increase traffic (including SEO)
- Ensure site is easily expandable / scalable for future features

**B. Speed of Results**

- Total project window: **220 hours (internship requirement)**
- Goal:
  - MVP (full website migration + login capability) completed **ASAP (~75% of total hours target)**
  - Remaining hours used for:
    - additional content updates
    - admin dashboard
    - refinements
- Existing Wix site remains **live during migration period**

**C. Content Ownership / Updates**

- MVP phase: **Nick updates all content**
- Post-MVP: Admin dashboard planned so **Rose + staff can manage content**

---

## SECTION 2 — Current System Inventory

**A. Domain Registrar Ownership**

- Believed to be **Wix (for now)**
- Potential transfer later depending on cost / strategy

**B. DNS Control**

- Assumed to be **Wix until domain transfer occurs**

**C. Wix Account Ownership**

- Remains **Rose only**

**D. Calendly Ownership**

- Remains unchanged

---

## SECTION 3 — Content + Site Scope

**Architecture Direction (Migration Improvement)**

- Current Wix site behaves like a **single‑page scroll experience (SPA‑like)**
- Migration will intentionally convert structure into:
  - **true routed multi‑page architecture (e.g., React + Router)**
  - improved **SEO indexability**
  - improved **future scalability for features (course / gated content / expansion)**

**A. Site Structure**

- Current site is effectively **single-page layout**

**B. Content Changes**

- Most content expected to remain the same
- Possible minor design improvements / layout options
- Existing palette + branding already business-aligned

**C. Media Asset Ownership**

- Rose owns and has access to all:
  - logos
  - photos
  - testimonials
  - branding assets

---

## SECTION 4 — Additional Features (Post-MVP Focus)

**A. Course Feature (not full learning platform)**

- Planned after MVP
- Early assumptions:
  - **Login required**
  - **Primarily gated content**
  - Payment may be required later (must design system so payment can be added cleanly)
  - Progress tracking considered **advanced / future enhancement**

**Important Scope Note**

- Future features are anticipated but **not included in MVP scope**

**B. Newsletter System**

- To be defined post‑MVP

**C. Blogging**

- Not likely in MVP scope
- Revisit later

---

## SECTION 5 — Hosting + Maintenance Expectations

- Target uptime: **As close to 100% as reasonably possible**
- Downtime response: **ASAP**
- Updates / enhancements: **case-by-case basis**
- Dependency updates:
  - Nick initially
  - Future maintainer possible
- Infrastructure costs:
  - Paid by Rose (if applicable)
- Repository ownership:
  - **Rose owns repository**
- Same ownership philosophy applies to:
  - hosting accounts
  - AWS accounts
  - third‑party tools

---

## SECTION 6 — Budget Reality

- Target annual cost: **$250–300/year total infrastructure goal**
- Strategic intent:
  - Do not significantly exceed current Wix cost baseline
- Hosting direction:
  - Prefer **Netlify free tier (if viable)**
  - Possible minimal AWS usage (low traffic assumption — especially year 1)
- SaaS tools only if clearly needed (decision after MVP)
- Preference:
  - low recurring cost
  - yearly billing where possible

---

## SECTION 7 — Timeline Constraints

- Project limited to **220 internship hours**
- No urgent launch pressure since:
  - tax season completed
  - current Wix site remains operational
